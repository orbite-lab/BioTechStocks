# Biotech Valuation Model — Claude Context

This file is the master context for any Claude agent working in this repo.
Read it first.

## What this project is

A biotech valuation and portfolio-sizing tool. Static HTML + JSON configs,
deployed via GitHub Pages. Two main pages:

- `index.html` — screener across ~45 companies, with filters, the Market
  Explorer, Catalyst Asymmetry Map, Portfolio Scenario Simulator, and the
  Portfolio Kelly Optimizer + Catalyst-Aware Kelly Schedule.
- `model.html` — per-company 5-scenario valuation model (mega_bear → bear
  → base → bull → psychedelic_bull), with editable TAM/SOM/PoS/APR/pen
  sliders, a Portfolio Kelly card, and a Valuation vs Peers card.

No build step. No framework. React via babel-standalone in `model.html`,
vanilla JS in `index.html`. Data is all JSON files.

## Core concepts

### Two-tier market per indication

Every indication has both:

- **TAM (`market.regions`)** — whole disease market. Identical across companies
  competing in the same area. Source of truth: `scripts/data/authoritative_regions_*.py`.
  Run `scripts/ops/reconcile_sliders.py` to propagate.
- **SOM (`market.company_slice`)** — this drug's capture. Unique per config.
  `reachPct × wtpPct × priceK × regions[rk].patientsK` across US/EU/ROW.

**Invariants (enforced by audit):**
- `SOM ≤ TAM` (always)
- `SOM ≥ salesM` for commercial drugs (peak can't be below current run rate)
- Scenario TPs monotonic: mega_bear ≤ bear ≤ base ≤ bull ≤ psychedelic_bull

### Pseudo-areas (`_*` prefix)

Areas starting with `_` are **meta-tags, not real disease classifications**.
Used for assets that don't fit the disease taxonomy:
- Platform / discovery indications without a chosen lead disease
- Pre-clinical research spanning multiple potential indications
- Pure technology investments where forcing a disease tag would mislead

Currently in use:
- `_platform.adc_discovery` (6990 Daiichi ADC platform R&D)
- `_platform.bicycle_discovery` (BCYC discovery pipeline)

**Behavior:**
- **Market Explorer** skips `_*` paths (clean disease drill-down)
- **Technology Explorer** shows them (classified by modality, where platforms belong)
- **validate.py** exempts from depth requirement and "missing TAM data" warning
  (still errors if `area` is entirely missing)
- **Audit** doesn't flag (no SOM/TAM math forced)
- **Display labels** in `data/taxonomy.json display_names`

**When to retire a `_*` tag**: once the asset picks a lead indication, retag to the
proper disease L3 (`oncology.lung.nsclc_driver.her2` etc.) and populate market.

### Modality taxonomy + display labels

Each asset has a `modality` field with `L1.L2.L3` path. Auto-aggregated into
`data/taxonomy.json modalities` by `scripts/recurring/rebuild_taxonomy.py`.

16 L1 modality classes (each represents a structurally distinct production class):
1. `small_molecule` — organic synthesis (kinase, GPCR, ion-channel, PPI, transporter, etc.)
2. `peptide` — solid-phase synthesis (incretins, hormones, cyclic peptides)
3. `antibody` — mAb / bispecific / fragment / fusion (Eylea is `antibody.fusion.vegf_trap`)
4. `adc` — antibody-drug conjugates (DXd-linker, multi-payload, peptide-drug)
5. `recombinant_protein` — non-antibody therapeutic proteins (cytokines, enzymes, toxins)
6. `plasma_protein` — blood plasma fractionation derivatives (IVIG, albumin, antithrombin, FVIII)
7. `biological_extract` — animal/plant tissue extracts (porcine surfactant Curosurf, betulin Filsuvez, heparin)
8. `cell_therapy` — autologous / allogeneic / in vivo CAR-T (NK, TCR slots reserved)
9. `gene_therapy` — DNA delivery one-shot (AAV, lentivirus reserved)
10. `gene_editing` — CRISPR / base / prime editing (own L1, distinct production from AAV)
11. `nucleic_acid` — RNA-modulating (ASO, siRNA, splice modulator, mRNA reserved)
12. `radiopharmaceutical` — radioligand / radioconjugate / alpha-emitter
13. `vaccine` — protein subunit / VLP / mRNA reserved
14. `xenotransplant` — gene-edited porcine organs (kidney, heart)
15. `formulation_modifier` — long-acting depot, half-life extender, ocular sustained, etc.
16. `aesthetic_or_other` — hyaluronic filler, etc.

**Convention notes:**
- Mechanism (agonist/antagonist) lives in L3, never as a sister L2 to structural class
- Kinase inhibitors are technically enzymes but kept as their own L2 by pharma convention
  (~1000 kinases, distinct ATP-pocket mechanism, distinct competitive landscape)
- BCL-2 / MDM2-p53 / KRAS-SOS1 are **PPI** targets (`protein_protein_inhibitor`),
  not enzyme inhibitors

**Display labels** (`data/taxonomy.json.display_names`): human-readable UI strings
for L1/L2/L3 keys. Used by index.html (Market+Tech Explorer) and model.html (asset
chips). Lookup priority: exact match → last-segment match → underscore-replace
fallback. Preserved across `rebuild_taxonomy.py` runs. Bulk-add via
`scripts/archive/_add_display_names.py`.

### Scenarios

Five: `mega_bear`, `bear`, `base`, `bull`, `psychedelic_bull`. Each has:
- `wt`: probability weight (0-100, should sum to 100)
- `val`: valuation params — `mult`, `dr`, `cannib` (non-SOTP) or
  `pipelineDR`, `pipelineMult`, `commercialMult`, `commercialRevM`, `milestones` (SOTP)
- `assumptions`: per-asset.indication `{pos, apr, pen}` — probability of success,
  approval prob rate, and peak revenue multiplier

### SOTP vs non-SOTP

- **SOTP** (`pipelineDR` present): commercial valued at `commercialRevM × commercialMult`,
  pipeline valued at `pipelineRA × pipelineMult / (1+pipelineDR)^n`. Used for mega-caps
  with both commercial + pipeline (e.g., LLY).
- **Non-SOTP**: single bucket, all assets contribute to `totalRA`, valued at
  `netRA × mult / (1+dr)^7`. Used for pure-pipeline or single-franchise companies.

### Fractional Kelly

- `model.html` per-stock: solves `f* = argmax Σ p_i ln(1 + f·r_i)` over the
  5-scenario distribution. Half Kelly (75% of growth, 50% of volatility) is the
  recommended sizing.
- `index.html` portfolio: multi-asset Kelly with taxonomy-based correlation
  (same-L3 ρ=0.70, same-L2 ρ=0.45, same-L1 ρ=0.25, sector-beta ρ=0.15),
  5000-sample Monte Carlo, projected gradient ascent with per-name + per-L1
  caps.
- Catalyst-Aware Kelly: structural baseline + Gaussian-kernel bumps around
  each catalyst date. `catInfluence` slider blends between pure steady-state
  and pure event-driven.

## How to add a new company

1. Copy an existing config as a template (e.g., `configs/VRDN.json` for a
   pure-pipeline; `configs/ABBV.json` for a big-pharma SOTP).
2. Fill in `company.{ticker, name, currentPrice, sharesOut, cash, currency}`.
3. List assets with their stage + indications. Each indication needs an `area`
   tag matching `taxonomy.json`.
4. Set initial `market.regions` manually **only** if the L3 area isn't already
   in `scripts/data/authoritative_regions_*.py`. Otherwise leave regions empty
   and run `py scripts/ops/reconcile_sliders.py` to auto-populate.
5. Fill `market.company_slice` per region (reachPct, wtpPct, priceK) — this
   is unique to this drug.
6. Add `salesM` if the drug is commercial (from latest 10-K/20-F).
7. Add catalysts with `{date, dateSort, asset, indication, title, type,
   fail_pos, fail_apr, success_pos, success_apr}`.
8. Add scenario weights + val + assumptions per scenario.
9. Add ticker to `configs/manifest.json`.
10. Run `py scripts/ops/audit_configs.py` — must be 0 CRIT before commit.

### If adding a new market (no authoritative entry)

Add the L3 area to the correct `scripts/data/authoritative_regions_*.py`
module with curated patientsK / wtpPct / priceK + sources. Then run
`py scripts/ops/reconcile_sliders.py` to populate all configs.

## Conventions

- **Currency**: `company.currency` is local (USD, EUR, JPY, HKD, etc.).
  Prices, cash, revenues displayed in local. FX applied when computing
  USD-normalized market cap or EV (see `FX_TO_USD` in `index.html`).
- **Numbers**: `patientsK` in thousands, `priceK` in thousands of USD
  (so `patientsK × priceK` = $M). `salesM`, `commercialRevM`, `cash` in $M.
  `sharesOut` in millions.
- **Dates**: ISO format in `dateSort` (for sorting); human-readable in `date`
  ("Q1 2026", "H2 2026").
- **Private companies** (delisted / wholly-owned subsidiaries / family
  conglomerates like Servier, Boehringer, Mitsubishi Tanabe): set
  `company.private: true` and use placeholder `currentPrice: 1`,
  `sharesOut: 1`, `yahooTicker: ""`. Add a `privateNote` field
  describing why no equity is tradeable. Audit skips price-relative
  flags; snapshot excludes them; `fetch_prices.py` skips the feed. The
  config still contributes to taxonomy / Market Explorer / Technology
  Explorer / drug-target indexes -- the point is to track drugs and
  competitive landscape without forcing meaningless TPs.
- **Encoding**: UTF-8 everywhere. Avoid em-dashes (--) in Python source on
  Windows — use `-- ` or `—` via `u"\u2014"`.

## Common operations

```
# Validate schema (blocks PR if any config violates the structural schema)
py scripts/ops/validate_schema.py

# Audit all configs -- should be 0 CRIT
py scripts/ops/audit_configs.py

# Scenario regression test: fail if any TP drifts >=10% from baseline
py scripts/ops/snapshot_scenarios.py
# After an intentional model/input change, regenerate and commit baseline:
py scripts/ops/snapshot_scenarios.py --update

# Schema + taxonomy validation (stricter than audit)
py scripts/recurring/validate.py

# Apply curated epi data + fix invariants
py scripts/ops/reconcile_sliders.py

# Recalibrate mega-cap scenarios (mature pharma tuning)
py scripts/ops/recalibrate_mega_cap.py           # dry-run
py scripts/ops/recalibrate_mega_cap.py --write   # apply

# Generate TAM/SOM/sales coverage report
py scripts/ops/tam_report.py

# Rebuild taxonomy.json from area tags across all configs
py scripts/recurring/rebuild_taxonomy.py
```

## Guardrails

1. **Never commit with `CRIT > 0`** from `audit_configs.py`. The audit
   GitHub Action blocks this on PRs to main.
2. **`data/` JSON files are fetched by HTML at `data/<name>.json`**
   (`data/financials.json`, `data/taxonomy.json`, `data/sector_benchmarks.json`,
   `data/prices.json`, `data/options_cache.json`). If you ever move them,
   update every `fetch("data/...")` in `index.html` and `model.html`, plus
   the scripts that write them and the GHA workflows that stage them.
3. **Never manually edit `market.regions`** for an area that has an
   authoritative entry — edit the Python module instead. The auto-sync
   would clobber hand edits on the next `reconcile_sliders` run.
4. **SOTP vs non-SOTP is sticky**: `recalibrate_mega_cap.py` detects
   `pipelineDR` in the val block and skips override — don't switch structures
   without intent.
5. **Disease taxonomy is pure clinical** — never tag technology as a disease
   sub-segment (no `myeloma.car_t`, no `cll_nhl.bcl2`). Technology lives in the
   modality taxonomy. Use `_*` pseudo-areas for true platform/discovery assets
   that can't pick a lead disease.
6. **Multi-indication assets** (Imbruvica, Brukinsa, Dupixent, Darzalex, etc.)
   should split `salesM` across their actual approved indications as separate
   indication entries, not lumped into one.
7. **Commit reconcile drift in the same commit as the feature**. When
   `reconcile_sliders.py` runs it walks every config and may rewrite
   `company_slice` values on tickers other than the one you're working on
   (whenever authoritative regions or salesM cap-recalibration produces a
   small numeric drift). Always `git add configs/` (the whole dir) and
   include the drift in the same commit -- never leave it as uncommitted
   working-tree noise. Otherwise the next `git pull --rebase` requires a
   stash dance because GHA bots will have committed the same drift to
   remote in parallel. Same rule for `data/targets.json` rebuilds.
