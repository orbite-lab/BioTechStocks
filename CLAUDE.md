# Biotech Valuation Model — Claude Context

This file is the master context for any Claude agent working in this repo.
Read it first.

## What this project is

A biotech valuation and portfolio-sizing tool. Static HTML + JSON configs,
deployed via GitHub Pages. Two main pages:

- `index.html` — screener across ~45 companies, with filters, the Market
  Explorer, Catalyst Asymmetry Map, Portfolio Scenario Simulator, and the
  Portfolio Kelly Optimizer + Catalyst-Aware Kelly Schedule.
- `model.html` — per-company 5-scenario valuation model (mega_bear → bear
  → base → bull → psychedelic_bull), with editable TAM/SOM/PoS/APR/pen
  sliders, a Portfolio Kelly card, and a Valuation vs Peers card.

No build step. No framework. React via babel-standalone in `model.html`,
vanilla JS in `index.html`. Data is all JSON files.

## Core concepts

### Two-tier market per indication

Every indication has both:

- **TAM (`market.regions`)** — whole disease market. Identical across companies
  competing in the same area. Source of truth: `scripts/data/authoritative_regions_*.py`.
  Run `scripts/ops/reconcile_sliders.py` to propagate.
- **SOM (`market.company_slice`)** — this drug's capture. Unique per config.
  `reachPct × wtpPct × priceK × regions[rk].patientsK` across US/EU/ROW.

**Invariants (enforced by audit):**
- `SOM ≤ TAM` (always)
- `SOM ≥ salesM` for commercial drugs (peak can't be below current run rate)
- Scenario TPs monotonic: mega_bear ≤ bear ≤ base ≤ bull ≤ psychedelic_bull

### Scenarios

Five: `mega_bear`, `bear`, `base`, `bull`, `psychedelic_bull`. Each has:
- `wt`: probability weight (0-100, should sum to 100)
- `val`: valuation params — `mult`, `dr`, `cannib` (non-SOTP) or
  `pipelineDR`, `pipelineMult`, `commercialMult`, `commercialRevM`, `milestones` (SOTP)
- `assumptions`: per-asset.indication `{pos, apr, pen}` — probability of success,
  approval prob rate, and peak revenue multiplier

### SOTP vs non-SOTP

- **SOTP** (`pipelineDR` present): commercial valued at `commercialRevM × commercialMult`,
  pipeline valued at `pipelineRA × pipelineMult / (1+pipelineDR)^n`. Used for mega-caps
  with both commercial + pipeline (e.g., LLY).
- **Non-SOTP**: single bucket, all assets contribute to `totalRA`, valued at
  `netRA × mult / (1+dr)^7`. Used for pure-pipeline or single-franchise companies.

### Fractional Kelly

- `model.html` per-stock: solves `f* = argmax Σ p_i ln(1 + f·r_i)` over the
  5-scenario distribution. Half Kelly (75% of growth, 50% of volatility) is the
  recommended sizing.
- `index.html` portfolio: multi-asset Kelly with taxonomy-based correlation
  (same-L3 ρ=0.70, same-L2 ρ=0.45, same-L1 ρ=0.25, sector-beta ρ=0.15),
  5000-sample Monte Carlo, projected gradient ascent with per-name + per-L1
  caps.
- Catalyst-Aware Kelly: structural baseline + Gaussian-kernel bumps around
  each catalyst date. `catInfluence` slider blends between pure steady-state
  and pure event-driven.

## How to add a new company

1. Copy an existing config as a template (e.g., `configs/VRDN.json` for a
   pure-pipeline; `configs/ABBV.json` for a big-pharma SOTP).
2. Fill in `company.{ticker, name, currentPrice, sharesOut, cash, currency}`.
3. List assets with their stage + indications. Each indication needs an `area`
   tag matching `taxonomy.json`.
4. Set initial `market.regions` manually **only** if the L3 area isn't already
   in `scripts/data/authoritative_regions_*.py`. Otherwise leave regions empty
   and run `py scripts/ops/reconcile_sliders.py` to auto-populate.
5. Fill `market.company_slice` per region (reachPct, wtpPct, priceK) — this
   is unique to this drug.
6. Add `salesM` if the drug is commercial (from latest 10-K/20-F).
7. Add catalysts with `{date, dateSort, asset, indication, title, type,
   fail_pos, fail_apr, success_pos, success_apr}`.
8. Add scenario weights + val + assumptions per scenario.
9. Add ticker to `configs/manifest.json`.
10. Run `py scripts/ops/audit_configs.py` — must be 0 CRIT before commit.

### If adding a new market (no authoritative entry)

Add the L3 area to the correct `scripts/data/authoritative_regions_*.py`
module with curated patientsK / wtpPct / priceK + sources. Then run
`py scripts/ops/reconcile_sliders.py` to populate all configs.

## Conventions

- **Currency**: `company.currency` is local (USD, EUR, JPY, HKD, etc.).
  Prices, cash, revenues displayed in local. FX applied when computing
  USD-normalized market cap or EV (see `FX_TO_USD` in `index.html`).
- **Numbers**: `patientsK` in thousands, `priceK` in thousands of USD
  (so `patientsK × priceK` = $M). `salesM`, `commercialRevM`, `cash` in $M.
  `sharesOut` in millions.
- **Dates**: ISO format in `dateSort` (for sorting); human-readable in `date`
  ("Q1 2026", "H2 2026").
- **Encoding**: UTF-8 everywhere. Avoid em-dashes (--) in Python source on
  Windows — use `-- ` or `—` via `u"\u2014"`.

## Common operations

```
# Audit all configs -- should be 0 CRIT
py scripts/ops/audit_configs.py

# Schema + taxonomy validation (stricter than audit)
py scripts/recurring/validate.py

# Apply curated epi data + fix invariants
py scripts/ops/reconcile_sliders.py

# Recalibrate mega-cap scenarios (mature pharma tuning)
py scripts/ops/recalibrate_mega_cap.py           # dry-run
py scripts/ops/recalibrate_mega_cap.py --write   # apply

# Generate TAM/SOM/sales coverage report
py scripts/ops/tam_report.py

# Rebuild taxonomy.json from area tags across all configs
py scripts/recurring/rebuild_taxonomy.py
```

## Guardrails

1. **Never commit with `CRIT > 0`** from `audit_configs.py`. The audit
   GitHub Action blocks this on PRs to main.
2. **Never move root JSON files** (`financials.json`, `taxonomy.json`,
   `sector_benchmarks.json`, `prices.json`, `options_cache.json`) without
   also updating every `fetch("...")` in `index.html` and `model.html`.
3. **Never manually edit `market.regions`** for an area that has an
   authoritative entry — edit the Python module instead. The auto-sync
   would clobber hand edits on the next `reconcile_sliders` run.
4. **SOTP vs non-SOTP is sticky**: `recalibrate_mega_cap.py` detects
   `pipelineDR` in the val block and skips override — don't switch structures
   without intent.
